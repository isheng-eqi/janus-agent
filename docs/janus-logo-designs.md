# Janus ASCII 太极图设计方案

> 为 Janus README 设计的 ASCII/Unicode 太极图（Taiji / Yin-Yang），3 个候选方案。
> 太极的阴阳两面 = Janus 双面神的过去/未来两面。

---

## 设计约束

- 纯 ASCII/Unicode 字符，无 emoji
- 大气醒目——README 顶部的第一视觉焦点
- 契合 Janus 双面神主题：阴阳 = 过去/未来、输入/输出、审视/决策
- 尺寸：15-25 行高，30-55 列宽
- 可用字符：◐ ◑ ◈ ◆ ◇ ○ ● ◎ ╭ ╮ ╰ ╯ ─ │ ┌ └ ░ ▒ ▓ █

---

## 方案一：乾坤太极图（The Primal Taiji）

**方向**：经典太极——用 ▓/░ 渐变块构建阴阳鱼，圆形边框 + 清晰 S 曲线 + 双鱼眼。

```
              ╭───────────────────────────────╮
            ╱                                   ╲
          ╱                                       ╲
         │             ▓▓▓▓▓▓▓▓▓▓▓▓▓              │
         │          ▓▓▓             ▓▓▓            │
         │        ▓▓                   ▓▓          │
         │       ▓                       ▓         │
         │      ▓                         ▓        │
         │      ▓                         ▓        │
         │      ▓           ○             ▓        │
         │      ▓                         ░        │
         │       ▓                       ░         │
         │        ░          ╳          ░          │
         │        ░                    ░           │
         │        ░         ●           ░          │
         │         ░                  ░            │
         │          ░░              ░░             │
         │            ░░░░░░░░░░░░░░               │
          ╲                                       ╱
            ╲                                   ╱
              ╰───────────────────────────────╯
```

**尺寸**：21 行 × 43 列

**设计解读**：

- **▓ 暗面（阴/过去）**：左上区域，用 ▓ 块填充——过去是已知的、确定的、厚重的
- **░ 亮面（阳/未来）**：右下区域，用 ░ 块填充——未来是半透明的、开放的、可塑的
- **S 曲线**：▓ → ░ 的过渡带就是 Janus 的决策边界——Gatekeeper 在此判断"过去能教未来什么"
- **○ 阳中之阴**：暗面中一颗白点——过去的经验中有尚未发现的盲区
- **● 阴中之阳**：亮面中一颗黑点——未来的不确定性中有确定的锚点
- **╳ 中心**：过去与未来交汇的瞬间——Janus 的 Planner 站在此刻做决策
- **圆形边框**：╭╮╰╯ 圆角 + ─ │ 直边——完整的循环，时间无尽

**优点**：最经典、识别度最高、任何人一眼认出太极；尺寸大气不做作
**缺点**：S 曲线用单个 ╳ 表达，细节不如矢量图

## 方案二：爻变太极（The Hexagram Taiji）

**方向**：像素艺术——用 █ 和 ░ 交替排列构成三角渐变，形似《易经》爻变之象。

```
           ╭─────────────────────────────╮
          ╱                               ╲
         │    █ █ █ █ █ █ █ █ █ █ █ █     │
         │     █ █ █ █ █ █ █ █ █ █ █      │
         │      █ █ █ █ █ █ █ █ █ █       │
         │       █ █ █ █ █ █ █ █ █        │
         │        █ █ █ █ █ █ █ █         │
         │         █ █ █ █ █ █ █          │
         │          █ █ ○ █ █ █           │
         │           █ █ █ █ █            │
         │            █ █ █ █             │
         │             █ ╳ █              │
         │            ░ ░ ░ ░             │
         │           ░ ░ ░ ░ ░            │
         │          ░ ░ ● ░ ░ ░           │
         │         ░ ░ ░ ░ ░ ░ ░          │
         │        ░ ░ ░ ░ ░ ░ ░ ░         │
         │       ░ ░ ░ ░ ░ ░ ░ ░ ░        │
         │      ░ ░ ░ ░ ░ ░ ░ ░ ░ ░       │
         │     ░ ░ ░ ░ ░ ░ ░ ░ ░ ░ ░      │
          ╲                               ╱
           ╰─────────────────────────────╯
```

**尺寸**：21 行 × 33 列

**设计解读**：

- **爻（yáo）**：《易经》六十四卦的基本单位——阴爻（⚋）和阳爻（⚊）
- **三角渐变**：顶部全 █（纯阳）→ 底部全 ░（纯阴）——这不是二元对立，而是光谱渐变
- **Janus 的决策**不是二选一，而是在阴阳光谱上找一个平衡点——这正是 Gatekeeper Tree 的哲学
- **█ █ 交替排列**像卦象的爻线——古代智慧与现代 AI 架构的跨时空对话
- **╳ 中心线**：阴阳在此交替、力量在此转换——Planner 的轴心
- **像素美学**：故意保留"数码感"，契合 AI agent 的技术本质

**优点**：独特性极高，不像任何常见的太极 ASCII art；爻变隐喻深；像素风格自带技术气质
**缺点**：识别为"太极"需要一点认知跳跃；对不熟悉易经的读者可能看不出爻的隐喻

## 方案三：铭文太极印（The Inscribed Taiji Seal）

**方向**：太极图顶部 + JANUS 铭文底部——合二为一的"封印"式设计，完整 README header。

```
               ╭───────────────────────────────────╮
             ╱                                       ╲
           ╱                                           ╲
          │               ▓▓▓▓▓▓▓▓▓▓▓▓▓                │
          │            ▓▓▓             ▓▓▓              │
          │          ▓▓                   ▓▓            │
          │         ▓                       ▓           │
          │        ▓                         ▓          │
          │       ▓                           ▓         │
          │       ▓             ○             ▓         │
          │       ▓                           ░         │
          │        ▓                         ░          │
          │         ▓           ╳           ░           │
          │          ░                     ░            │
          │          ░          ●          ░            │
          │           ░                   ░             │
          │            ░░               ░░              │
          │              ░░░░░░░░░░░░░░░                │
           ╲                                           ╱
             ╲                                       ╱
               ╰───────────────────────────────────╯
               ╔═══════════════════════════════════╗
               ║     J  A  N  U  S                ║
               ║     ═══════════════════          ║
               ║     The Two-Faced Agent          ║
               ║                                  ║
               ║     past  ◈  future              ║
               ╚═══════════════════════════════════╝
```

**尺寸**：25 行 × 43 列

**设计解读**：

- **上部太极**：和方案一同款经典太极，作为图形标识
- **下部铭文**：双线框（╔╗╚╝）像一枚印章——JANUS 被"封印"在阴阳之下
- **◈ 分隔 past/future**：菱中带圆 = 既是门也是眼——Janus 在此凝视两个方向
- **双框嵌套**：外层图形（太极）+ 内层文字（铭文）= Gatekeeper + Planner 的双层结构
- **═ 分隔线**：将项目名与 tagline 分开——视觉层次清晰
- **留白行**：铭文框内有一行空白——呼吸感，让信息不拥挤

**优点**：最完整——图形 + 文字 + tagline 一气呵成，可以直接作为 README 的 header 粘贴使用；印章造型有东方仪式感
**缺点**：尺寸最大（25 行），在小屏幕上可能占满首屏；信息密度高，不适合只想放一个纯图形的场景

## 最终推荐：方案一 — 乾坤太极图 ⭐

**理由**：

1. **一眼太极**——识别度无可匹敌。放在 README 顶部，任何读者不需要解释就知道"这是太极图、这是阴阳"
2. **尺寸克制**——21 行 × 43 列，在 README 的大标题下方不会过分膨胀，留出足够的后续阅读空间
3. **留白呼吸**——圆形内部有大量留白，▓ 块和 ░ 块各司其职，视觉不拥挤，符合"黑白基调、简洁有力"的要求
4. **哲学准确**——○ 在暗面（过去中的未知）、● 在亮面（未来中的确定性）、╳ 在正中（此刻的决策），三项精确对应 Janus 三层时间观
5. **品牌友好**——这个太极图可以脱离 JANUS 文字独立使用，缩小后做 favicon、社交头像依然可辨
6. **经典永不过时**——相比方案二的实验性、方案三的场景限定，方案一在任何上下文都成立

### 各方案适用场景

- **选方案一**：你需要一个经典的图形标识，太极 = Janus 的视觉锚点（推荐）
- **选方案二**：你想强调 Janus 的"技术框架"属性，《易经》爻变的隐喻让项目有思想深度
- **选方案三**：你要一个"粘贴即用"的完整 README header，图形 + 文字一次性搞定
